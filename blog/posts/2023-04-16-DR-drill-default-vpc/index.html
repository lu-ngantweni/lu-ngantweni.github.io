Why is my DRS Drill/Recovery Instance launching in default VPC?

16 Apr 2023 in posts

Understanding why DRS drill and recovery instances launch in the default VPC and how to resolve VPCIdNotSpecified errors.

I have come across a few cases where customers launch recovery instances and need further guidance on understanding the following.

  1. The Problem
  2. Locating the Launch Template
  3. Why This Happens
  4. The Fix

The Problem

  • Why is it launching in default VPC
  • Why is it failing to launch due to no default VPC or subnet specified (VPCIdNotSpecified)

DRS uses EC2 launch templates to launch drill and recovery EC2 instances, these launch templates are created automatically with default settings to launching drill/recovery instances. These are created upon installation of the AWS replication agent.

Locating the Launch Template

Generally the launch templates are available on EC2 dashboard. For ease of use we can access the launch templates directly from DRS console by selecting source –> Launch Settings –> Launch Template.

DRS console path to launch template settings.

By default the subnet is configured to “Don’t include in launch template”.

Default launch template subnet setting.

Why This Happens

The above configuration will be why customer drill/recovery instance launches:

  • In default VPC, if one exists
  • Fail to launch with “An error occurred (VPCIdNotSpecified) when calling the RunInstances operation: No default VPC for this user” should default VPC not exist.

VPCIdNotSpecified error when no default VPC exists.

From the documentation:

Customers that do not have a default VPC must modify the EC2 Launch Template and explicitly define the subnet in which to launch. Failure to do so will result in errors when launching Drill or Recovery instances.

Subnet - Choose the subnet. This is the subnet within which the network interface is located and the Drill or Recovery instance will be launched. Elastic Disaster Recovery selects the default VPC subnet by default (if one exists).

The Fix

To workaround/fix the issue we can modify the Launch Template and specify subnet.

Modifying the launch template to specify a subnet.

These settings can be found under the “Network Settings”.

Network settings section in the launch template.

If a customer does not explicitly set the subnet in the EC2 Launch template this will result in drill/recovery instances launching in default VPC or fail with error if non-exists.
