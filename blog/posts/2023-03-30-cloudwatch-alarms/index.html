Create CloudWatch Alarms for Multiple EC2 Instances & EBS Volumes [Script]

30 Mar 2023 in posts

This article shares some scripts to simplify the creation cloudwatch alarms for multiple EC2 instances and EBS volumes.

I come across a few cases whereby customers are looking for a simplified approach to create cloudwatch alarms for more than 1 resource (e.g I have 30 instances how can I create an alarm for all of them). The below are actually 1 script but can be modified to serve different resources (EBS or EC2) additionally these can serve different metrics below are just examples but you can edit parameters of the put-metric-alarm cli to your use case.

EBS Alarms

The example script below creates a throughput alarm for all volumes in af-south-1 region

#!/bin/bash
listVolumes='/tmp/listvolumes.txt'
volumes=$(aws ec2 describe-volumes --query "Volumes[*].[VolumeId]" --filters Name=tag:Environment,Values=Test --output text --region af-south-1)
echo ${volumes} > ${listVolumes}
for Volumes in `cat ${listVolumes}`
do    
        echo $Volumes    
        aws cloudwatch put-metric-alarm \    
        --alarm-name ${Volumes} \    
        --alarm-description "Alarm when EBS volume exceeds 100MB throughput" \    
        --metric-name VolumeReadBytes \    
        --namespace AWS/EBS \    
        --statistic Average \    
        --period 300 \    
        --threshold 100000000 \    
        --comparison-operator GreaterThanThreshold \    
        --dimensions Name=VolumeId,"Value=${Volumes}" \    
        --evaluation-periods 3 \    
        --alarm-actions arn:aws:sns:af-south-1:[Account ID]:my-alarm-topic
done

EC2 Alarms

The example script below creates a CPUutilization alarm that alarms when CPU reaches 70% [Can be filtered based on preferences]

#!/bin/bash
listInstances='/tmp/Instances.txt'
instances=$(aws ec2 describe-instances --region eu-west-2 --query "Reservations[*].Instances[*].[InstanceId]" --filters Name=instance-state-name,Values=running --output text)
echo ${instances} > ${listInstances}
for Instance in `cat ${listInstances}`
do
	 echo $Instance
	 aws cloudwatch put-metric-alarm \
        --region eu-west-2\
        --alarm-name status-check-${Instance} \
        --alarm-description "Alarm when instance experience status check fail" \
        --metric-name "StatusCheckFailed_Instance" \
        --namespace "AWS/EC2" \
        --statistic "Sum" \
        --period 300 \
        --threshold 0 \
        --comparison-operator "GreaterThanThreshold"  \
        --dimensions "Name=InstanceId","Value=${Instance}" \
        --evaluation-periods 1 \
        --alarm-actions "arn:aws:sns:eu-west-2:[Account ID]:StatusCheckFailed" \
        --unit Count
done
