Why DRS/MGN agent failed to install due to “No space left on device” but the volume has sufficient disk space.

20 Apr 2024 in posts

This article covers a recent issue I encountered whilst assisting customer with troubleshooting “No space left on device” error on AL2023 when sufficient disk space was available.

  1. Problem Statement
  2. Troubleshooting

Problem Statement

During installation you will see standard failure when executing agent installation:

Downloading the AWS Replication Agent onto the source server...

Error: Failed Downloading the AWS Replication Agent
Installation failed.
Learn more about installation issues in our documentation at https://docs.aws.amazon.com/drs/latest/userguide/Troubleshooting-Agent-Issues.html

You can review install logs for context on failure, by default the installation log is saved in the install directory

$ sudo cat aws_replication_agent_installer.log [Truncated]

Error: Failed Downloading the AWS Replication Agent
Installation failed.
Learn more about installation issues in our documentation at https://docs.aws.amazon.com/drs/latest/userguide/Troubleshooting-Agent-Issues.html  [installation_id: 4495e699-4c21-4eee-bac6-c1f6b79ee6db, agent_version: 5.19.0.2023.352.1140, aws_instance_id: i-08907809e46fda714, mac_addresses: 16445040878151, _origin_client_type: installer]
2024-01-13 14:02:22,825 ERROR      Installation was not finished successfully
Traceback (most recent call last):
  File "dirrus/installer_shared/installer_main.py", line 451, in download_installer_files
  File "shared/installer_utils/file_utils.py", line 23, in save_files
OSError: [Errno 28] No space left on device

Troubleshooting

Based on the error we confirm the available disk space

$ df -hT

Filesystem     Type      Size  Used Avail Use% Mounted on
devtmpfs       devtmpfs  4.0M     0  4.0M   0% /dev
tmpfs          tmpfs     979M     0  979M   0% /dev/shm
tmpfs          tmpfs     392M  5.4M  386M   2% /run
/dev/xvda1     xfs       8.0G  1.6G  6.5G  20% /
tmpfs          tmpfs     300M     0  300M   0% /tmp
/dev/xvda128   vfat       10M  1.3M  8.7M  13% /boot/efi
tmpfs          tmpfs     196M     0  196M   0% /run/user/1000

From the output we can see that our root directory / is 20% utilized with sufficient space to accommodate the agent. However our /tmp directory is mounted on tmpfs and currently does not have sufficient space to accommodate agent based on documentation [1] the DRS agent requires at-least 500MB to install successfully. tmpfs is a temporary filesystem that resides in memory and/or swap partition(s). By default, a tmpfs partition has its maximum size set to half of the available RAM on the system, however it is possible to overrule this value.

Therefore the resolve the issue you can proceed with the following steps:

  1. Temporary override the default value you can proceed with the following command to accommodate the agent 
    mount -o remount,size=1G /tmp
  2. To permanently override this value you can add below entry in /etc/fstab based on desired size 
    tmpfs   /tmp         tmpfs   rw,nodev,nosuid,size=1G          0  0
  3. Depending on whether this is production or testing EC2 instance customer can increase their instance size which in turn allow more memory assignment to tmpfs

After performing method:1 we now see that our tmpfs directory is 1GB

df -hT
Filesystem     Type      Size  Used Avail Use% Mounted on
devtmpfs       devtmpfs  4.0M     0  4.0M   0% /dev
tmpfs          tmpfs     979M     0  979M   0% /dev/shm
tmpfs          tmpfs     392M  5.4M  386M   2% /run
/dev/xvda1     xfs       8.0G  1.6G  6.5G  20% /
tmpfs          tmpfs     1.0G     0  1.0G   0% /tmp
/dev/xvda128   vfat       10M  1.3M  8.7M  13% /boot/efi
tmpfs          tmpfs     196M     0  196M   0% /run/user/1000

We now have a successful agent installation

Source Network already exists for vpc: vpc-0e74d42255ebfc9d9.
The following is the source server ID: s-341d64cdb11d39ef2.
The AWS Replication Agent was successfully installed.

[1] Installation requirements for AWS Replication Agent [2] tmpfs
